# Как писать посты в Word

Ты пишешь в Word → присылаешь мне `.docx` → я конвертирую скриптом за ~30 секунд. Эта инструкция — как готовить Word-файлы, чтобы скрипт их правильно разобрал.

---

## Базовые правила

### Один пост = один `.docx`

Имя файла — любое («Книга Кэрса.docx», «Про фокус.docx»). Slug задам при конвертации.

### Структура документа

```
ЗАГОЛОВОК ПОСТА (первая строка документа, любой стиль)

Первый абзац текста.

ПОДЗАГОЛОВОК (стиль Heading 2)

Ещё текст с **жирным** и *курсивом*.

• Элемент списка 1 (стиль List Bullet)
• Элемент списка 2
• Элемент списка 3

> Цитата (стиль Quote)

Текст после цитаты.

Теги: книги, продукт, дизайн
```

---

## Что скрипт понимает идеально

| Что в Word | Во что превратится |
|---|---|
| Первая строка документа | Заголовок поста (любой стиль — Heading 1, Heading 3, обычный; не дублируется в тексте) |
| Стиль **Heading 2, 3** | Подзаголовки в тексте |
| **Жирный** (Cmd+B) | **жирный** в markdown |
| *Курсив* (Cmd+I) | *курсив* в markdown |
| [Ссылки](url) (Cmd+K) | Кликабельные ссылки |
| Списки (стиль List Bullet) | Маркированный список |
| Стиль **Quote** или **IntenseQuote** | Блок цитаты |
| Картинки (Insert → Pictures) | Вытаскиваются, ресайзятся, вставляются в текст |
| Строка «Теги: слово, слово» | Поле tags в frontmatter |

## Что скрипт понимает плохо

Скрипт **предупредит** если обнаружит:

- **Таблицы** — превратятся в отдельные абзацы без табличной структуры. Лучше: перевести в текст или вставить скриншотом
- **Колонки** — потеряются, текст станет одной колонкой
- **Сноски** — исчезнут. Лучше: перевести в скобки прямо в тексте
- **SmartArt, Excel-объекты, диаграммы** — не экспортируются. Лучше: скриншот
- **Отслеживание изменений** — если есть непринятые правки, может быть путаница. Прими/отклони все через Review → Accept All
- **Комментарии** — проигнорируются

## Картинки

### Как вставлять

**Правильно**: Insert → Pictures → Picture from File. Word сохраняет оригинал без пережатия.

**Неправильно**: copy-paste из браузера. Word может пережать изображение в низкое качество.

### Куда

Прямо в Word-документ, в том месте, где картинка должна быть в посте. Можно в любом размере — в вёрстке всё равно растянется на ширину текстовой колонки.

### Подписи

Если нужна подпись — напиши её курсивом на следующей строке после картинки:

```
[картинка]
*Фото автора, 2025*
```

### Качество

Word Mac по умолчанию может сжимать картинки. Чтобы этого не было:

**При сохранении файла:**
- File → Save As
- Внизу окна сохранения — кнопка **Compress Pictures** или **Reduce File Size**
- Выбрать **Original resolution** + **Best for printing** и убрать галочку «Apply to all»

**Проверить что не сжато:**
- Размер `.docx`-файла > размер исходных фото. Если `.docx` весит 200KB, а ты вставила 3 фото по 2MB каждая — Word их пережал

Если сомневаешься — пришли мне, я проверю разрешение и скажу.

## Теги

Три рабочих способа — любой на выбор:

### А. Строкой в конце документа

```
Теги: книги, философия, мышление
```

### Б. Хэштегами в конце

```
#книги #философия #мышление
```

### В. Просто в сообщении мне

«Вот пост, теги: книги, философия, мышление»

Все три дают одинаковый результат — теги попадают в frontmatter и становятся кликабельными кнопками под заголовком.

## Что я спрошу при получении .docx

Если это непонятно из файла:

1. **Slug на латинице** для URL (короткий английский перевод, не транслит). Придумаю сам, если не пришлёшь
2. **Язык** — ru или en (по умолчанию ru)
3. **Рубрика (kicker)** — если нужна подпись над заголовком (например «Книги», «Мысли»)
4. **Закрепить на главной?** — да/нет (по умолчанию нет)
5. **Это черновик?** — если да, пост не появится в ленте до публикации

Дата — всегда сегодняшняя, summary не нужно — лидом идёт первое предложение поста.

## Локальный запуск скрипта (если захочешь сама)

```bash
# обязательно slug, остальное опционально
python3 scripts/word-to-md.py "my-post.docx" --slug my-post

# с полными параметрами
python3 scripts/word-to-md.py "my-post.docx" \
  --slug finite-games \
  --date 2026-04-21 \
  --lang ru \
  --kicker "Книги" \
  --pin

# dry-run — показать результат, ничего не создавая
python3 scripts/word-to-md.py "my-post.docx" --slug test --dry-run
```

Скрипт создаёт:
- `content/posts/<slug>.md` — сам пост
- `assets/posts/<slug>-1.jpg`, `-2.jpg`, … — картинки

После этого:
```bash
python3 scripts/build.py   # локальная пересборка (необязательно — Actions всё равно сделает)
git add .
git commit -m "add: <название поста>"
git push
```

Через минуту пост на сайте.
