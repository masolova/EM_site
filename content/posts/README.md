# Как написать пост

## Коротко

1. Скопируй `_template.md` → переименуй в `<slug>.md` (например, `finite-games.md`).
2. Заполни frontmatter (верхний блок между `---`).
3. Напиши текст ниже frontmatter в Markdown.
4. Закоммить в репозиторий. GitHub Action сам соберёт сайт.

**Имя файла = slug + `.md`.** Без дат в имени — дата живёт только во frontmatter.

## Как конвертировать из Word

Если пишешь в Word, мы вместе:
1. Ты присылаешь .docx.
2. Я конвертирую текст в Markdown, ставлю правильный frontmatter, сохраняю в `content/posts/`.
3. Ты проверяешь результат и коммитишь.

## Обязательные поля во frontmatter

| Поле | Что это | Пример |
|------|---------|--------|
| `title` | Заголовок статьи | `"Конечные и бесконечные игры"` |
| `slug` | Английский slug для URL | `"finite-games"` |
| `date` | Дата ПЕРВОЙ публикации (не меняется) | `"2026-04-20"` |
| `lang` | Язык (пока только `ru`) | `"ru"` |

Лид для карточек и мета-тегов = первое предложение тела поста (вытягивается автоматически). Поля `summary` нет.

## Опциональные поля

| Поле | Что это | По умолчанию |
|------|---------|--------------|
| `tags` | Список тегов | пусто |
| `kicker` | Подпись над заголовком | пусто |
| `reading_time` | Время чтения в минутах | считается автоматически |
| `status` | `draft` или `published` | `published` |
| `pinned` | Закреплён ли на главной | `false` |

## Как править уже опубликованный пост

1. Открой `content/posts/<slug>.md`.
2. Правь текст.
3. **Не трогай поле `date`** — оно остаётся датой первой публикации, по ней сортируется лента.
4. Закоммить. Пост **не поднимется** в ленте как новый, никаких пометок «обновлено» на странице не появляется — правка проходит молча.

## Правила для slug

- Только латиница в нижнем регистре
- Слова через дефис
- Коротко (2–4 слова)
- Примеры: `finite-games`, `ai-search`, `product-dna`, `real-estate-mvp`
- Не обязан быть дословным переводом заголовка — можно короткую тематическую метку

## Что умеет Markdown

- `**жирный**` → **жирный**
- `*курсив*` → *курсив*
- `[ссылка](https://example.com)` → [ссылка](https://example.com)
- `## Заголовок 2`, `### Заголовок 3`
- Списки через `-` или `1.`
- Цитаты через `>`
- Картинки: `![описание](путь/к/картинке.jpg)`
